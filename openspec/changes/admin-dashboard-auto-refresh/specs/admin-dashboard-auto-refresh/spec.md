## 新增需求

### 需求:管理员可以在总览页启用自动刷新
系统必须允许管理员在 `/admin/dashboard` 启用或关闭自动刷新，并必须允许管理员从预设刷新间隔中选择当前刷新周期。首次访问或本地无历史配置时，自动刷新必须默认关闭。

#### 场景:首次访问默认关闭
- **当** 管理员首次打开 `/admin/dashboard`
- **并且** 浏览器中不存在该页面的自动刷新配置
- **那么** 系统必须以“自动刷新关闭”状态展示页面
- **并且** 系统不得在未显式开启前自动触发周期刷新

#### 场景:启用 30 秒自动刷新
- **当** 管理员在 `/admin/dashboard` 启用自动刷新
- **并且** 管理员选择 30 秒作为刷新间隔
- **那么** 系统必须保存该启用状态和 30 秒间隔
- **并且** 系统必须开始显示对应的自动刷新倒计时

#### 场景:关闭自动刷新
- **当** 管理员主动关闭 `/admin/dashboard` 的自动刷新
- **那么** 系统必须停止倒计时
- **并且** 系统不得继续触发后续周期刷新

### 需求:自动刷新必须基于当前筛选条件刷新总览数据
系统必须在自动刷新触发时，使用当前页面已选的 `startDate`、`endDate` 和 `granularity` 重新加载总览统计、趋势图、用户趋势和消费排行数据。

#### 场景:自动刷新沿用当前日期范围与粒度
- **当** 管理员已将 `/admin/dashboard` 的日期范围设置为指定起止日期
- **并且** 当前粒度为 `hour`
- **并且** 自动刷新倒计时归零
- **那么** 系统必须使用当前 `startDate`、`endDate` 和 `granularity=hour` 重新请求仪表盘数据
- **并且** 刷新结果必须同时更新统计卡片、图表和排行区域

#### 场景:筛选变更后后续自动刷新使用新条件
- **当** 管理员修改日期范围或粒度
- **那么** 系统必须先按新筛选条件刷新页面数据
- **并且** 此后自动刷新必须继续沿用更新后的筛选条件

### 需求:系统必须持久化自动刷新偏好并在成功刷新后重置倒计时
系统必须在浏览器本地持久化 `/admin/dashboard` 的自动刷新启用状态和刷新间隔。系统每次成功执行手动刷新、筛选触发刷新或自动刷新后，都必须重新开始下一轮倒计时。

#### 场景:刷新页面后恢复自动刷新配置
- **当** 管理员此前已经为 `/admin/dashboard` 启用了自动刷新并选择了预设间隔
- **并且** 管理员刷新浏览器页面或重新进入该页面
- **那么** 系统必须恢复此前保存的启用状态与刷新间隔
- **并且** 系统必须从该间隔重新开始倒计时

#### 场景:手动刷新后重置倒计时
- **当** `/admin/dashboard` 已开启自动刷新
- **并且** 管理员手动点击刷新按钮
- **那么** 系统必须立即执行一次刷新
- **并且** 在本次刷新成功后把自动刷新倒计时重置为当前间隔值

### 需求:系统禁止自动刷新触发重叠请求
当 `/admin/dashboard` 已存在进行中的整页刷新任务时，系统禁止再次启动新的自动刷新请求。

#### 场景:上一次自动刷新尚未完成
- **当** `/admin/dashboard` 已开启自动刷新
- **并且** 上一次整页刷新仍在进行中
- **并且** 自动刷新倒计时再次归零
- **那么** 系统必须跳过本次自动刷新触发
- **并且** 系统不得并发发起第二轮相同数据请求
