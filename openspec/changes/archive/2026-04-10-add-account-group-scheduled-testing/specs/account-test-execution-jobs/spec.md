## 新增需求

### 需求:管理员可以发起分组立即测试任务
系统必须允许管理员在分组管理操作区点击“测试账号（立即）”创建测试任务，并返回任务标识与初始状态。

#### 场景:创建立即测试任务
- **当** 管理员在分组管理操作区点击“测试账号（立即）”并确认执行
- **那么** 系统必须创建后台测试任务并返回任务标识
- **并且** 任务初始状态必须为“待执行”或“执行中”

### 需求:任务并发上限必须为 5 且补位执行
系统必须限制同一测试任务的并发测试账号数量不超过 5；当某账号测试完成后，系统必须从队列中补位下一个账号继续执行，直到队列为空。

#### 场景:分组账号超过 5 个
- **当** 某测试任务的账号数量超过 5
- **那么** 系统必须同时最多执行 5 个账号测试
- **并且** 任一账号测试完成后必须立即补位下一个账号继续执行

### 需求:系统必须提供实时日志与进度
系统必须为测试任务提供实时日志或进度流，日志需包含账号级别的开始、完成与结果状态。

#### 场景:查看实时日志
- **当** 管理员在操作区点击“实时日志”并打开某任务
- **那么** 系统必须实时输出该任务的进度与账号测试结果

### 需求:任务状态必须可查询
系统必须提供任务状态查询能力，至少包含待执行、执行中与已完成。

#### 场景:查询任务状态
- **当** 管理员请求某任务的状态
- **那么** 系统必须返回该任务的当前状态
