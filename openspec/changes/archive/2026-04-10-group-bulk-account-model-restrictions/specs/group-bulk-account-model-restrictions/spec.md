## 新增需求

### 需求:分组批量设置必须以当前分组下账号为作用范围
系统必须将分组管理中的模型限制批量设置作用于当前分组绑定的账号集合，而不是用户集合。

#### 场景:分组下存在多个账号
- **当** 管理员在分组管理中对某个分组执行模型限制批量设置
- **那么** 系统必须以该分组当前绑定的账号作为更新对象
- **并且** 返回命中的账号数量

#### 场景:分组下没有账号
- **当** 管理员对一个没有绑定账号的分组执行模型限制批量设置
- **那么** 系统必须返回明确提示说明当前分组没有可更新账号

### 需求:批量设置修改的是账号本体配置
系统必须将分组页提交的模型限制写入账号本身；如果账号同时属于多个分组，更新后该账号在其他分组中的使用也必须体现新配置。

#### 场景:账号同时属于多个分组
- **当** 管理员在分组 A 中批量修改了某个同时属于分组 A 和分组 B 的账号模型限制
- **那么** 系统必须保存该账号的新模型限制配置
- **并且** 该账号在分组 B 中继续被使用时也必须体现该新配置
