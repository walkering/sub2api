## 新增需求

### 需求:分组管理必须支持按分组批量设置账号模型限制
系统必须允许管理员在分组管理中对当前分组下的账号批量设置模型白名单或模型映射，并将结果写入账号已有的模型限制配置。

#### 场景:管理员为分组内账号批量设置白名单
- **当** 管理员在分组管理中选择某个分组并提交模型白名单配置
- **那么** 系统必须将该配置批量写入该分组下的账号
- **并且** 账号后续必须按现有账号模型白名单语义生效

#### 场景:管理员为分组内账号批量设置模型映射
- **当** 管理员在分组管理中选择某个分组并提交模型映射配置
- **那么** 系统必须将该映射批量写入该分组下的账号
- **并且** 账号后续必须按现有账号模型映射语义生效

### 需求:分组入口必须与账号批量编辑的模型限制语义一致
系统必须让分组管理入口与账号批量编辑中的模型限制行为保持一致，包括模式切换、白名单转映射写法、映射冲突校验和禁用提示。

#### 场景:分组页提交冲突映射
- **当** 管理员在分组页提交存在冲突的模型映射规则
- **那么** 系统必须拒绝保存
- **并且** 返回与账号批量编辑一致的校验提示
