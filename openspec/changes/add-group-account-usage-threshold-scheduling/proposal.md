## 为什么

当前分组调度只能基于账号可用性、限流和已有窗口费用规则筛选账号，缺少“账号使用量接近上限时主动退出新调度”的能力。运营需要按分组设置一个统一的使用量百分比阈值，在账号接近配额上限时及时止损，同时又不能粗暴打断正在进行的会话。

## 变更内容

- 为分组新增账号使用量调度阈值配置，按百分比控制账号何时停止接收新的调度。
- 当账号使用量达到或超过阈值时，账号必须退出新的分配候选，但若该账号仍承载当前活跃会话，则继续服务该会话直到自然结束。
- 活跃会话结束后，系统必须立即停止该账号在所属分组中的调度，并在用户后续请求时改选同组内其他可用账号。
- 为管理端提供阈值配置、展示与校验规则，并明确无配额上限账号的处理方式。

## 功能 (Capabilities)

### 新增功能
- `group-account-usage-threshold-scheduling`: 分组可配置账号使用量阈值，并据此控制账号的新调度、粘性续用和阈值后摘除行为。

## 影响

- 后端：分组模型与管理接口需要新增阈值字段；调度快照、网关粘性续用与活跃会话判断需要纳入该规则。
- 前端：分组管理页需要新增阈值配置项、说明文案与校验反馈。
- 调度行为：达到阈值的账号将从新请求候选集中移除，但允许当前未结束的对话继续完成。
- 文档与测试：需要补充分组阈值规则、边界场景和调度回退验证。
