## 新增需求

### 需求:管理员可以为分组创建定时测试计划
系统必须允许管理员在账号管理或分组管理入口为指定分组创建定时测试计划，计划必须包含分组、时间规则、批次大小与偏移量。

#### 场景:从账号管理创建定时测试
- **当** 管理员在账号管理点击“定时测试”并选择分组、时间规则、批次大小与偏移量
- **那么** 系统必须创建该分组的定时测试计划

#### 场景:从分组管理创建定时测试
- **当** 管理员在分组管理操作区点击“定时测试”并填写计划参数
- **那么** 系统必须创建该分组的定时测试计划

### 需求:定时测试必须按批次与偏移执行
系统必须在计划触发时按稳定顺序确定分组账号列表，并按 `batch_size` 分批执行，批次启动间隔必须为 `offset`。

#### 场景:批次错峰执行
- **当** 某计划触发且分组内账号数量大于 `batch_size`
- **那么** 系统必须先启动首批测试
- **并且** 在 `offset` 时间后启动下一批，直到全部批次完成

### 需求:触发时无可测试账号必须给出明确结果
系统必须在计划触发时检测可测试账号数量，若为 0 必须标记为“无可测试账号”并不创建测试批次。

#### 场景:分组为空
- **当** 某计划触发且分组内没有任何可测试账号
- **那么** 系统必须标记本次触发为无可测试账号
- **并且** 系统不得创建任何测试批次

### 需求:计划参数必须校验
系统必须校验计划参数合法性，`batch_size` 必须大于 0，`offset` 必须大于或等于 0。

#### 场景:参数非法
- **当** 管理员提交 `batch_size` 小于等于 0 或 `offset` 小于 0 的计划参数
- **那么** 系统必须拒绝创建或更新计划并返回错误提示
