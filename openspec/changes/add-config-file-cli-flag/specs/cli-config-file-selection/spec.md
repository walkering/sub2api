## 新增需求

### 需求:显式配置文件参数
系统必须支持在启动 Sub2API 时通过命令行参数显式指定配置文件路径，且至少支持 `-c <path>` 形式。

#### 场景:使用短参数指定配置文件
- **当** 管理员执行 `./sub2api -c config.yaml`
- **那么** 系统必须读取 `config.yaml` 作为本次启动的配置文件来源

#### 场景:使用长参数指定配置文件
- **当** 管理员执行 `./sub2api -config /etc/sub2api/custom.yaml`
- **那么** 系统必须读取 `/etc/sub2api/custom.yaml` 作为本次启动的配置文件来源

### 需求:显式配置文件优先级
当显式传入配置文件路径时，系统必须优先使用该文件，而不是继续从默认搜索路径中挑选其他 `config.yaml`。

#### 场景:显式路径覆盖默认搜索路径
- **当** 当前目录和 `/etc/sub2api` 下都存在默认配置文件，且管理员执行 `./sub2api -c ./alt.yaml`
- **那么** 系统必须优先读取 `./alt.yaml`
- **并且** 系统禁止静默改为读取默认搜索路径中的其他配置文件

#### 场景:未传入显式路径时保持原行为
- **当** 管理员执行不带 `-c` 的现有启动命令
- **那么** 系统必须继续按照现有默认搜索路径与环境变量逻辑加载配置

### 需求:启动链路一致性
系统必须让 setup 阶段、bootstrap 配置读取、正式配置读取与监听地址解析使用同一显式配置文件选择结果。

#### 场景:setup 模式使用同一配置文件
- **当** 管理员执行 `./sub2api -setup -c ./config.yaml`
- **那么** setup 阶段必须使用 `./config.yaml` 解析监听地址和启动所需配置

#### 场景:正式启动使用同一配置文件
- **当** 管理员执行 `./sub2api -c ./config.yaml`
- **那么** bootstrap 配置读取与正式运行配置读取必须基于同一份 `./config.yaml`

### 需求:显式路径错误可观测
当管理员显式指定的配置文件不存在、不可读或格式非法时，系统必须返回明确错误并终止启动。

#### 场景:配置文件不存在
- **当** 管理员执行 `./sub2api -c ./missing.yaml`
- **那么** 系统必须返回配置文件读取失败的错误
- **并且** 系统禁止静默回退到默认搜索路径继续启动

#### 场景:配置文件格式错误
- **当** 管理员执行 `./sub2api -c ./broken.yaml` 且文件内容不是合法 YAML
- **那么** 系统必须返回配置解析错误
- **并且** 系统禁止继续进入正常启动流程
