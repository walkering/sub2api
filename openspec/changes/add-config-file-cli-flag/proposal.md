## 为什么

当前服务端只能按固定搜索路径读取 `config.yaml`，运行命令无法显式指定配置文件位置。对于二进制部署、同机多实例、临时验证和从发布包目录直接启动的场景，管理员需要像 `./sub2api -c config.yaml` 这样直接指定配置文件，避免依赖工作目录、`DATA_DIR` 或系统路径推断。

## 变更内容

- 为服务端运行命令新增 `-c <path>` / `--config <path>` 风格的配置文件参数，用于显式指定配置文件。
- 启动阶段与正常运行阶段统一支持该参数：包括 setup 模式、bootstrap 配置读取、正式启动配置读取，以及获取监听地址等轻量配置读取场景。
- 当显式传入配置文件路径时，优先使用该文件；未传入时保持现有按默认搜索路径加载配置的行为不变。
- 补充 CLI 帮助文案、部署文档与测试，明确示例命令 `./sub2api -c config.yaml`。

## 功能 (Capabilities)

### 新增功能
- `cli-config-file-selection`: 允许管理员在启动 Sub2API 时通过命令行参数显式指定配置文件路径，并在启动链路中以该路径作为最高优先级配置来源。
