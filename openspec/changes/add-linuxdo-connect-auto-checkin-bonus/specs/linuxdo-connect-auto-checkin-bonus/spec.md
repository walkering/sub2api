## 新增需求

### 需求:管理员可以开启或关闭 LinuxDo Connect 自动签到奖励
系统必须允许管理员在系统设置中开启或关闭 LinuxDo Connect 登录自动签到奖励。该开关默认必须为关闭，关闭时系统必须保持现有 LinuxDo Connect 登录行为，不自动签到、不发放金额奖励。

#### 场景:读取默认关闭的自动签到奖励开关
- **当** 系统初始化默认设置且管理员尚未修改 LinuxDo Connect 自动签到奖励配置
- **那么** 系统必须返回 `linuxdo_connect_auto_checkin_bonus_enabled=false`

#### 场景:管理员开启自动签到奖励
- **当** 管理员在系统设置中将 LinuxDo Connect 自动签到奖励开关设为开启并保存
- **那么** 系统必须保存该配置
- **并且** 后续 LinuxDo Connect 登录成功流程必须开始执行自动签到检查

#### 场景:管理员关闭自动签到奖励
- **当** 管理员在系统设置中将 LinuxDo Connect 自动签到奖励开关设为关闭并保存
- **那么** 系统必须保存该配置
- **并且** 后续 LinuxDo Connect 登录成功流程必须不执行自动签到奖励逻辑

### 需求:开启后 LinuxDo Connect 登录成功必须自动执行每日签到奖励
当 `linuxdo_connect_auto_checkin_bonus_enabled=true` 时，系统必须在 LinuxDo Connect 登录成功后自动尝试执行签到奖励。单个本地用户在同一服务端自然日内最多只能成功一次；首次成功时系统必须随机赠送 `1-5` 的余额金额。

#### 场景:直接 LinuxDo OAuth 登录当天首次签到成功
- **当** LinuxDo Connect 自动签到奖励开关已开启
- **并且** 用户通过 LinuxDo OAuth 直接成功登录
- **并且** 该用户在当前服务端自然日尚未获得 LinuxDo 自动签到奖励
- **那么** 系统必须在登录成功流程中自动完成签到
- **并且** 系统必须随机赠送 `1-5` 的余额金额

#### 场景:邀请码补全注册当天首次签到成功
- **当** LinuxDo Connect 自动签到奖励开关已开启
- **并且** 用户在邀请码模式下完成 `complete-registration` 并成功创建本地账号
- **并且** 该用户在当前服务端自然日尚未获得 LinuxDo 自动签到奖励
- **那么** 系统必须在登录成功流程中自动完成签到
- **并且** 系统必须随机赠送 `1-5` 的余额金额

#### 场景:同一用户当天重复通过 LinuxDo Connect 登录
- **当** LinuxDo Connect 自动签到奖励开关已开启
- **并且** 用户已经在当前服务端自然日获得过一次 LinuxDo 自动签到奖励
- **并且** 该用户再次通过 LinuxDo Connect 成功登录
- **那么** 系统必须完成登录流程
- **并且** 系统必须不再次发放自动签到奖励

#### 场景:开关关闭时 LinuxDo Connect 登录成功
- **当** LinuxDo Connect 自动签到奖励开关为关闭
- **并且** 用户通过 LinuxDo Connect 成功登录或完成邀请码补全注册
- **那么** 系统必须完成现有登录流程
- **并且** 系统必须不创建自动签到奖励记录
- **并且** 系统必须不增加用户余额

#### 场景:自动签到奖励执行异常导致登录失败
- **当** LinuxDo Connect 自动签到奖励开关已开启
- **并且** 用户本次登录原本应执行自动签到奖励
- **并且** 奖励记录写入、余额更新、事务提交或相关依赖发生非“当天已签到”异常
- **那么** 系统必须拒绝本次 LinuxDo Connect 登录成功结果
- **并且** 系统必须不返回成功登录令牌或成功登录响应
- **并且** 前端必须不能把这次请求视为普通登录成功

### 需求:系统必须将本次自动签到结果回传给前端
系统必须在 LinuxDo Connect 登录成功响应中携带本次自动签到结果，使前端能够区分“仅登录成功”和“登录成功且本次已签到奖励”。当本次自动签到成功时，前端必须能够获得实际奖励金额。

#### 场景:本次登录触发自动签到奖励
- **当** 用户本次 LinuxDo Connect 登录成功
- **并且** 系统本次实际完成自动签到并发放奖励
- **那么** 登录成功响应必须包含“本次已奖励”的结果标识
- **并且** 登录成功响应必须包含本次实际赠送的金额

#### 场景:本次登录未触发自动签到奖励
- **当** 用户本次 LinuxDo Connect 登录成功
- **并且** 自动签到奖励开关关闭或该用户当天已签到
- **那么** 登录成功响应必须明确表示“本次未奖励”
- **并且** 前端必须能够据此不展示签到成功奖励提示

### 需求:自动签到奖励必须独立于 LinuxDo Connect 赠送订阅能力工作
系统必须将 LinuxDo Connect 自动签到奖励视为独立能力。无论 LinuxDo Connect 首次注册赠送订阅能力是否已归档、是否已开启，本次登录是否执行自动签到奖励都必须仅由自动签到奖励开关、每日幂等状态和本次登录结果决定。

#### 场景:未启用或未归档赠送订阅能力时自动签到仍可生效
- **当** LinuxDo Connect 自动签到奖励开关已开启
- **并且** LinuxDo Connect 首次注册赠送订阅能力未启用、未归档或未配置
- **并且** 用户本次 LinuxDo Connect 登录满足自动签到奖励条件
- **那么** 系统必须仍然按照自动签到奖励规则执行签到与发奖
