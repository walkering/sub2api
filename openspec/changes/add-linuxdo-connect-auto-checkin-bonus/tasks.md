## 2. 自动签到奖励与持久化

- [x] 2.1 设计并实现 LinuxDo 自动签到奖励记录的持久化结构，支持按 `user_id + 日期 + 来源` 做唯一幂等控制，并记录实际奖励金额。
- [x] 2.2 在 LinuxDo Connect 登录成功流程中新增自动签到奖励服务逻辑；开关开启时，当天首次成功登录随机赠送 `1-5` 余额，并与余额更新放入同一事务。
- [x] 2.3 覆盖 LinuxDo OAuth 直接登录和邀请码补全注册两条成功路径，确保同一用户同一天重复登录不会重复奖励。

## 3. 登录返回契约与前端提示

- [x] 3.1 扩展 LinuxDo OAuth 回调重定向参数和 `completeLinuxDoOAuthRegistration` 响应，回传本次自动签到是否成功及实际奖励金额。
- [x] 3.2 更新 `LinuxDoCallbackView`，仅在本次自动签到成功时弹出签到成功提示并展示奖励金额，否则保持现有登录成功提示。

## 4. 测试

- [x] 4.1 增加后端测试，验证开关关闭不触发奖励、开启后首次登录奖励 `1-5`、同日重复登录不重复奖励，以及邀请码补全路径同样生效。
