## 新增需求

### 需求:管理员可以按分组导出账号
系统必须允许管理员在导出时选择一个或多个分组，仅导出这些分组内的账号。当未选择任何分组时，系统必须沿用现有导出行为导出全部账号。

#### 场景:按选定分组导出
- **当** 管理员选择分组 A、B 并点击导出
- **那么** 系统必须只导出属于分组 A、B 的账号
- **并且** 不得包含其他分组账号

#### 场景:未选择分组时导出全部
- **当** 管理员未选择任何分组并执行导出
- **那么** 系统必须导出全部账号

### 需求:管理员可以在导入时指定目标分组
系统必须允许管理员在导入时选择一个目标分组，导入成功的账号必须归入该分组。

#### 场景:导入到指定分组
- **当** 管理员选择目标分组 C 并导入账号文件
- **那么** 系统必须把所有成功导入的账号归入分组 C

### 需求:系统必须校验目标分组合法性
系统必须在导入前校验目标分组存在且管理员有权限访问，非法分组必须拒绝导入并返回错误。

#### 场景:目标分组不存在
- **当** 管理员选择不存在的分组并发起导入
- **那么** 系统必须拒绝导入
- **并且** 返回分组无效的错误

#### 场景:无权限访问分组
- **当** 管理员选择无权限的分组并发起导入
- **那么** 系统必须拒绝导入
- **并且** 返回权限不足的错误

### 需求:显式选择的目标分组优先生效
当导入文件包含分组字段且管理员选择了目标分组时，系统必须以目标分组为准，禁止使用文件内分组覆盖。

#### 场景:文件分组与目标分组冲突
- **当** 导入文件中某账号的分组字段为 A
- **并且** 管理员选择的目标分组为 B
- **那么** 该账号导入后必须归入分组 B
