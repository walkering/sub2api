## 新增需求

### 需求:管理员可以按数量迁移指定分组中的可用账号
系统必须允许管理员指定源分组、目标分组和迁移数量，把满足迁移资格的账号从源分组移动到目标分组。当可迁移账号数量不足时，系统必须拒绝整个迁移请求，禁止执行部分迁移。

#### 场景:成功迁移足量可用账号
- **当** 管理员选择源分组 A、目标分组 B，并请求迁移 5 个满足资格的账号
- **那么** 系统必须把 5 个账号从分组 A 移动到分组 B
- **并且** 这 5 个账号迁移后必须不再保留对分组 A 的原绑定

#### 场景:可迁移数量不足时拒绝迁移
- **当** 管理员请求从源分组迁移 10 个账号
- **并且** 实际满足迁移资格的账号少于 10 个
- **那么** 系统必须拒绝本次迁移
- **并且** 系统必须不移动任何账号

### 需求:系统必须在迁移前校验账号类型与运行状态
系统必须在服务端统一校验分组迁移候选账号的类型和运行状态。系统禁止迁移非同类型账号，也禁止迁移状态异常、处于限流中或当前正在使用的账号。

#### 场景:候选账号存在多个类型
- **当** 源分组中满足其他条件的候选账号包含多个 `type`
- **那么** 系统必须拒绝本次迁移
- **并且** 系统必须提示只能迁移同类型账号

#### 场景:候选账号包含不可迁移账号
- **当** 源分组中的部分账号处于错误、禁用、限流中或正在使用状态
- **那么** 系统必须把这些账号排除在可迁移集合之外
- **并且** 仅允许剩余满足资格的账号参与数量校验和迁移
