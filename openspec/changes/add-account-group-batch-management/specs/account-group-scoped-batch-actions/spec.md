## 新增需求

### 需求:管理员可以按分组范围执行批量账号操作
系统必须允许管理员在已勾选账号基础上进一步指定一个分组范围，仅对该分组内的勾选账号执行重置状态、刷新令牌和批量编辑。

#### 场景:按分组范围重置状态
- **当** 管理员已勾选多个账号
- **并且** 管理员为“重置状态”选择了分组 A
- **那么** 系统必须仅对当前同时属于分组 A 的勾选账号执行重置状态

#### 场景:按分组范围刷新令牌
- **当** 管理员已勾选多个账号
- **并且** 管理员为“刷新令牌”选择了分组 B
- **那么** 系统必须仅对当前同时属于分组 B 的勾选账号执行刷新令牌

#### 场景:分组范围内没有任何勾选账号
- **当** 管理员为批量操作选择了某个分组
- **并且** 当前勾选账号中没有任何账号属于该分组
- **那么** 系统必须拒绝执行该批量操作
- **并且** 系统必须提示该分组下没有可处理的已选账号

### 需求:按分组范围批量编辑时必须保持现有勾选式更新语义
系统必须在按分组范围执行批量编辑时继续沿用现有的字段勾选语义：只有被启用或勾选的字段才允许更新，未启用的字段必须保持原值不变。

#### 场景:仅更新已勾选字段
- **当** 管理员对分组 A 范围内的已选账号打开批量编辑
- **并且** 管理员只勾选了“状态”和“代理”两个字段
- **那么** 系统必须只更新分组 A 范围内账号的“状态”和“代理”
- **并且** 其他未勾选字段必须保持原值不变

#### 场景:设置目标分组与作用范围分组同时存在
- **当** 管理员对分组 A 范围内的已选账号执行批量编辑
- **并且** 管理员在批量编辑中勾选了“账号分组”并设置新的目标分组列表
- **那么** 系统必须将分组 A 仅作为本次操作的作用范围
- **并且** 系统必须把勾选字段中的目标分组列表作为实际写入值
